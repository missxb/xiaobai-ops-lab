## 三、自动化修复脚本

### 3.1 Pod 自动重启

```python
#!/usr/bin/env python3
"""
pod_auto_restart.py - Pod 自动重启服务
当检测到 Pod CrashLooping 时自动重启
"""

import json
import logging
from http.server import HTTPServer, BaseHTTPRequestHandler
import subprocess
import requests

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class WebhookHandler(BaseHTTPRequestHandler):
    def do_POST(self):
        content_length = int(self.headers['Content-Length'])
        post_data = self.rfile.read(content_length)
        data = json.loads(post_data.decode('utf-8'))
        
        # 处理告警
        for alert in data.get('alerts', []):
            if alert.get('status') == 'firing':
                self.handle_alert(alert)
            elif alert.get('status') == 'resolved':
                self.handle_resolution(alert)
        
        self.send_response(200)
        self.send_header('Content-type', 'application/json')
        self.end_headers()
        self.wfile.write(json.dumps({'status': 'ok'}).encode())
    
    def handle_alert(self, alert):
        alert_name = alert['labels'].get('alertname')
        namespace = alert['labels'].get('namespace', 'default')
        pod = alert['labels'].get('pod', '')
        
        logger.info(f"处理告警: {alert_name}, Pod: {namespace}/{pod}")
        
        if alert_name == 'K8sPodCrashLooping':
            self.restart_pod(namespace, pod)
        elif alert_name == 'K8sPodOOMKilled':
            self.increase_memory(namespace, pod)
        elif alert_name == 'K8sPodNotReady':
            self.check_pod_readiness(namespace, pod)
    
    def restart_pod(self, namespace, pod):
        """重启 Pod"""
        try:
            result = subprocess.run(
                ['kubectl', 'delete', 'pod', pod, '-n', namespace, '--grace-period=30'],
                capture_output=True, text=True, timeout=60
            )
            if result.returncode == 0:
                logger.info(f"Pod {namespace}/{pod} 已重启")
            else:
                logger.error(f"重启失败: {result.stderr}")
        except Exception as e:
            logger.error(f"重启异常: {e}")
    
    def increase_memory(self, namespace, pod):
        """增加 Pod 内存限制"""
        try:
            # 获取当前内存限制
            result = subprocess.run(
                ['kubectl', 'get', 'pod', pod, '-n', namespace, '-o', 'json'],
                capture_output=True, text=True
            )
            pod_info = json.loads(result.stdout)
            
            # 增加 50% 内存
            for container in pod_info['spec']['containers']:
                if 'resources' in container and 'limits' in container['resources']:
                    current = container['resources']['limits'].get('memory', '512Mi')
                    new_memory = self.scale_memory(current, 1.5)
                    subprocess.run(
                        ['kubectl', 'set', 'resources', 'pod', pod,
                         '-n', namespace,
                         f'--limits=memory={new_memory}',
                         f'--containers={container["name"]}'],
                        capture_output=True, text=True
                    )
                    logger.info(f"容器 {container['name']} 内存增加到 {new_memory}")
        except Exception as e:
            logger.error(f"增加内存失败: {e}")
    
    def scale_memory(self, memory_str, factor):
        """缩放内存值"""
        if memory_str.endswith('Gi'):
            value = float(memory_str[:-2])
            return f"{int(value * factor)}Gi"
        elif memory_str.endswith('Mi'):
            value = float(memory_str[:-2])
            return f"{int(value * factor)}Mi"
        return memory_str

if __name__ == '__main__':
    server = HTTPServer(('0.0.0.0', 8080), WebhookHandler)
    logger.info("自动化修复服务启动，监听端口 8080")
    server.serve_forever()
```

### 3.2 节点自动摘除

```bash
#!/bin/bash
# auto-cordon.sh - 节点自动摘除脚本
# 当节点 NotReady 超过 5 分钟自动摘除

set -euo pipefail

NODE=${1:?用法: $0 <node-name>}

echo "=== 节点自动摘除: ${NODE} ==="

# 检查节点状态
STATUS=$(kubectl get node ${NODE} -o jsonpath='{.status.conditions[?(@.type=="Ready")].status}')
LAST_TRANSITION=$(kubectl get node ${NODE} -o jsonpath='{.status.conditions[?(@.type=="Ready")].lastTransitionTime}')

# 计算 NotReady 持续时间
if [ "${STATUS}" = "False" ]; then
  NOT_READY_TIME=$(date -d "${LAST_TRANSITION}" +%s)
  CURRENT_TIME=$(date +%s)
  DURATION=$(( (CURRENT_TIME - NOT_READY_TIME) / 60 ))
  
  if [ "${DURATION}" -gt 5 ]; then
    echo "节点 NotReady 超过 5 分钟，执行摘除..."
    
    # 标记不可调度
    kubectl cordon ${NODE}
    
    # 驱逐 Pod
    kubectl drain ${NODE} \
      --ignore-daemonsets \
      --delete-emptydir-data \
      --force \
      --grace-period=60 \
      --timeout=300s
    
    echo "节点 ${NODE} 已摘除"
    
    # 发送告警
    curl -X POST https://oapi.dingtalk.com/robot/send?access_token=xxx \
      -H 'Content-Type: application/json' \
      -d "{
        \"msgtype\": \"text\",
        \"text\": {
          \"content\": \"⚠️ 节点 ${NODE} NotReady 超过 5 分钟，已自动摘除\"
        }
      }"
  else
    echo "节点 NotReady 不足 5 分钟，暂不处理"
  fi
else
  echo "节点状态正常"
fi
```

### 3.3 磁盘自动清理

```bash
#!/bin/bash
# auto-disk-cleanup.sh - 磁盘自动清理脚本

set -euo pipefail

THRESHOLD=${1:-80}  # 默认 80%

echo "=== 磁盘自动清理 ==="

# 获取磁盘使用率
USAGE=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')

if [ "${USAGE}" -gt "${THRESHOLD}" ]; then
  echo "磁盘使用率 ${USAGE}% 超过阈值 ${THRESHOLD}%"
  
  # 清理 Docker 镜像
  echo "清理未使用的 Docker 镜像..."
  docker image prune -f
  
  # 清理 Docker 容器
  echo "清理已停止的 Docker 容器..."
  docker container prune -f
  
  # 清理 Docker 卷
  echo "清理未使用的 Docker 卷..."
  docker volume prune -f
  
  # 清理日志
  echo "清理旧日志文件..."
  find /var/log -name "*.log" -mtime +7 -delete
  find /var/log -name "*.gz" -mtime +30 -delete
  
  # 清理 K8s 缓存
  echo "清理 K8s 缓存..."
  rm -rf /var/cache/kubernetes/*
  
  # 检查清理结果
  NEW_USAGE=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')
  echo "清理完成: ${USAGE}% -> ${NEW_USAGE}%"
  
  # 发送告警
  if [ "${NEW_USAGE}" -gt "${THRESHOLD}" ]; then
    curl -X POST https://oapi.dingtalk.com/robot/send?access_token=xxx \
      -H 'Content-Type: application/json' \
      -d "{
        \"msgtype\": \"text\",
        \"text\": {
          \"content\": \"⚠️ 磁盘自动清理后使用率仍为 ${NEW_USAGE}%，需要人工介入\"
        }
      }"
  fi
else
  echo "磁盘使用率 ${USAGE}% 正常"
fi
```

## 四、自动化部署脚本

```bash
#!/bin/bash
# auto-deploy.sh - 自动化部署脚本

set -euo pipefail

APP_NAME=${1:?用法: $0 <app-name>}
VERSION=${2:?用法: $0 <app-name> <version>}
NAMESPACE=${3:-production}

echo "=== 自动化部署: ${APP_NAME} v${VERSION} ==="

# 1. 检查镜像
echo "检查镜像..."
if ! docker manifest inspect harbor.example.com/${NAMESPACE}/${APP_NAME}:${VERSION} > /dev/null 2>&1; then
  echo "❌ 镜像不存在: harbor.example.com/${NAMESPACE}/${APP_NAME}:${VERSION}"
  exit 1
fi

# 2. 备份当前版本
echo "备份当前版本..."
CURRENT_VERSION=$(kubectl get deployment ${APP_NAME} -n ${NAMESPACE} \
  -o jsonpath='{.spec.template.spec.containers[0].image}' | cut -d: -f2)
echo "当前版本: ${CURRENT_VERSION}"

# 3. 滚动更新
echo "执行滚动更新..."
kubectl set image deployment/${APP_NAME} \
  ${APP_NAME}=harbor.example.com/${NAMESPACE}/${APP_NAME}:${VERSION} \
  -n ${NAMESPACE}

# 4. 等待部署完成
echo "等待部署完成..."
if kubectl rollout status deployment/${APP_NAME} -n ${NAMESPACE} --timeout=300s; then
  echo "✅ 部署成功"
  
  # 5. 健康检查
  echo "执行健康检查..."
  sleep 10
  HEALTH_STATUS=$(curl -s -o /dev/null -w "%{http_code}" http://${APP_NAME}.${NAMESPACE}.svc.cluster.local/health)
  
  if [ "${HEALTH_STATUS}" = "200" ]; then
    echo "✅ 健康检查通过"
  else
    echo "⚠️ 健康检查失败 (HTTP ${HEALTH_STATUS})"
    echo "执行回滚..."
    kubectl rollout undo deployment/${APP_NAME} -n ${NAMESPACE}
    echo "已回滚到版本 ${CURRENT_VERSION}"
  fi
else
  echo "❌ 部署超时"
  echo "执行回滚..."
  kubectl rollout undo deployment/${APP_NAME} -n ${NAMESPACE}
  echo "已回滚到版本 ${CURRENT_VERSION}"
fi
```

## 五、运维脚本库

```bash
#!/bin/bash
# ops-toolkit.sh - 运维工具集

set -euo pipefail

# 显示菜单
show_menu() {
  echo "=== 运维工具集 ==="
  echo "1. 集群健康检查"
  echo "2. Pod 资源使用"
  echo "3. 节点维护模式"
  echo "4. 部署/回滚"
  echo "5. 日志查看"
  echo "6. 进入 Pod"
  echo "7. 清理资源"
  echo "8. 备份 etcd"
  echo "0. 退出"
  echo ""
  read -p "选择: " choice
}

# 集群健康检查
health_check() {
  echo "=== 集群健康检查 ==="
  kubectl get nodes -o wide
  echo ""
  kubectl get pods -A --field-selector=status.phase!=Running,status.phase!=Succeeded 2>/dev/null || echo "所有 Pod 正常"
}

# Pod 资源使用
pod_resources() {
  echo "=== Pod 资源使用 ==="
  echo "CPU 使用 TOP 10:"
  kubectl top pods -A --sort-by=cpu 2>/dev/null | head -11
  echo ""
  echo "内存使用 TOP 10:"
  kubectl top pods -A --sort-by=memory 2>/dev/null | head -11
}

# 节点维护模式
node_maintenance() {
  read -p "节点名称: " NODE
  read -p "操作 (enter/exit): " ACTION
  
  if [ "${ACTION}" = "enter" ]; then
    echo "进入维护模式: ${NODE}"
    kubectl cordon ${NODE}
    kubectl drain ${NODE} --ignore-daemonsets --delete-emptydir-data --force
  elif [ "${ACTION}" = "exit" ]; then
    echo "退出维护模式: ${NODE}"
    kubectl uncordon ${NODE}
  fi
}

# 部署/回滚
deploy_rollback() {
  read -p "应用名称: " APP
  read -p "操作 (deploy/rollback): " ACTION
  
  if [ "${ACTION}" = "deploy" ]; then
    read -p "版本: " VERSION
    ./auto-deploy.sh ${APP} ${VERSION}
  elif [ "${ACTION}" = "rollback" ]; then
    read -p "命名空间 (default): " NS
    ./rollback.sh ${NS:-default} ${APP}
  fi
}

# 日志查看
view_logs() {
  read -p "Pod 名称: " POD
  read -p "命名空间 (default): " NS
  read -p "行数 (100): " LINES
  
  kubectl logs ${POD} -n ${NS:-default} --tail=${LINES:-100} -f
}

# 进入 Pod
exec_pod() {
  read -p "Pod 名称: " POD
  read -p "命名空间 (default): " NS
  
  kubectl exec -it ${POD} -n ${NS:-default} -- /bin/bash
}

# 清理资源
cleanup() {
  echo "=== 清理资源 ==="
  echo "1. 清理已删除命名空间的资源"
  echo "2. 清理失败的 Pod"
  echo "3. 清理旧的 ReplicaSet"
  echo "4. 清理未使用的 PVC"
  
  read -p "选择: " choice
  case $choice in
    1) kubectl get pods -A | grep Terminating | awk '{print $1, $2}' | xargs -r kubectl delete pod --force --grace-period=0 ;;
    2) kubectl delete pods --field-selector=status.phase=Failed -A ;;
    3) kubectl get rs -A | awk '$2 == 0 {print $1, $2}' | xargs -r kubectl delete rs ;;
    4) echo "未使用的 PVC:"; kubectl get pvc -A | grep -v Bound ;;
  esac
}

# 主循环
while true; do
  show_menu
  case $choice in
    1) health_check ;;
    2) pod_resources ;;
    3) node_maintenance ;;
    4) deploy_rollback ;;
    5) view_logs ;;
    6) exec_pod ;;
    7) cleanup ;;
    8) /opt/scripts/etcd-backup.sh ;;
    0) exit 0 ;;
    *) echo "无效选择" ;;
  esac
  echo ""
done
```

# playbooks/01-auto-remediation.yml
---
- name: 自动化故障修复
  hosts: all
  become: yes
  vars:
    max_cpu_usage: 80
    max_memory_usage: 85
    max_disk_usage: 80
  tasks:
    # 1. 检查并重启失败的服务
    - name: 检查服务状态
      systemd:
        name: "{{ item }}"
        state: started
        enabled: yes
      loop:
        - nginx
        - containerd
        - kubelet
      register: service_status
      ignore_errors: yes

    - name: 发送服务重启告警
      debug:
        msg: "服务 {{ item.item }} 已自动重启"
      loop: "{{ service_status.results }}"
      when: item.changed

    # 2. 清理磁盘空间
    - name: 清理日志文件
      find:
        paths: /var/log
        patterns: "*.log.*"
        age: 7d
        state: absent
      when: ansible_mounts | selectattr('mount', 'equalto', '/') | map(attribute='size_available') | first < (ansible_mounts | selectattr('mount', 'equalto', '/') | map(attribute='size_total') | first * (100 - max_disk_usage) / 100)

    # 3. 清理 Docker 资源
    - name: 清理未使用的 Docker 镜像
      command: docker image prune -f
      when: ansible_mounts | selectattr('mount', 'equalto', '/') | map(attribute='size_available') | first < (ansible_mounts | selectattr('mount', 'equalto', '/') | map(attribute='size_total') | first * (100 - max_disk_usage) / 100)

## 六、面试题与思考

### Q1: 如何实现运维的自动化和智能化？

**答：** 分三个阶段：

1. **自动化**：将重复性操作脚本化、工具化
   - 自动部署、自动扩缩、自动备份
   - 使用 Ansible/Terraform 等工具

2. **自愈化**：根据监控告警自动修复
   - Pod 自动重启
   - 节点自动摘除
   - 服务自动回滚

3. **智能化**：基于 AI/ML 预测和优化
   - 异常检测
   - 容量预测
   - 智能调度

### Q2: 如何设计一个高可用的运维平台？

**答：**

1. **冗余部署**：所有组件多副本部署
2. **故障隔离**：不同组件独立部署，避免级联故障
3. **降级策略**：核心功能优先保障，非核心功能可降级
4. **监控告警**：对运维平台自身进行监控
5. **备份恢复**：定期备份配置和数据

### Q3: 如何评估运维自动化的效果？

**答：** 关键指标：

1. **MTTR (Mean Time To Repair)**：平均修复时间
2. **MTBF (Mean Time Between Failures)**：平均故障间隔
3. **部署频率**：每天/每周部署次数
4. **变更失败率**：部署后导致故障的比例
5. **人工干预次数**：需要人工介入的告警比例
