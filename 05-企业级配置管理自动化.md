## 三、系统初始化 Playbook

```yaml
# playbooks/01-system-init.yml
---
- name: 系统初始化
  hosts: all
  become: yes
  vars:
    timezone: "Asia/Shanghai"
    ntp_servers:
      - ntp.aliyun.com
      - ntp1.aliyun.com

  tasks:
    # ===== 基础配置 =====
    - name: 设置主机名
      hostname:
        name: "{{ inventory_hostname }}"

    - name: 添加 hosts 记录
      lineinfile:
        path: /etc/hosts
        line: "{{ hostvars[item].ansible_host }} {{ item }}"
      loop: "{{ groups['all'] }}"

    - name: 设置时区
      timezone:
        name: "{{ timezone }}"

    # ===== 内核参数 =====
    - name: 设置内核参数
      sysctl:
        name: "{{ item.key }}"
        value: "{{ item.value }}"
        sysctl_file: /etc/sysctl.d/99-custom.conf
        reload: yes
      loop: "{{ sysctl_params | dict2items }}"

    # ===== 安全配置 =====
    - name: 关闭 SELinux
      selinux:
        state: disabled
      when: ansible_os_family == "RedHat"
      ignore_errors: yes

    - name: 关闭防火墙
      systemd:
        name: firewalld
        state: stopped
        enabled: no
      ignore_errors: yes

    # ===== 安装基础软件 =====
    - name: 安装基础软件包
      yum:
        name: "{{ base_packages }}"
        state: present
        update_cache: yes
      when: ansible_os_family == "RedHat"

    # ===== 创建 ansible 用户 =====
    - name: 创建 ansible 用户
      user:
        name: ansible
        shell: /bin/bash
        groups: wheel
        append: yes
        create_home: yes

    - name: 配置 SSH 密钥
      authorized_key:
        user: ansible
        key: "{{ lookup('file', '~/.ssh/id_rsa.pub') }}"
        state: present

    # ===== 配置 SSH =====
    - name: 配置 SSH 安全
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: "{{ item.regexp }}"
        line: "{{ item.line }}"
      loop:
        - { regexp: '^#?PermitRootLogin', line: 'PermitRootLogin no' }
        - { regexp: '^#?PasswordAuthentication', line: 'PasswordAuthentication no' }
        - { regexp: '^#?MaxAuthTries', line: 'MaxAuthTries 3' }
        - { regexp: '^#?ClientAliveInterval', line: 'ClientAliveInterval 300' }
        - { regexp: '^#?ClientAliveCountMax', line: 'ClientAliveCountMax 3' }
      notify: restart sshd

    # ===== 配置日志 =====
    - name: 配置日志轮转
      copy:
        dest: /etc/logrotate.d/custom
        content: |
          /var/log/*.log {
              daily
              rotate 30
              compress
              delaycompress
              missingok
              notifempty
              create 640 root root
              sharedscripts
              postrotate
                  /bin/kill -HUP `cat /var/run/syslogd.pid 2> /dev/null` 2> /dev/null || true
              endscript
          }

  handlers:
    - name: restart sshd
      systemd:
        name: sshd
        state: restarted
```

## 四、安全加固 Playbook

```yaml
# playbooks/02-security-hardening.yml
---
- name: 安全加固
  hosts: all
  become: yes

  tasks:
    # ===== 密码策略 =====
    - name: 设置密码复杂度
      lineinfile:
        path: /etc/security/pwquality.conf
        regexp: "{{ item.regexp }}"
        line: "{{ item.line }}"
      loop:
        - { regexp: '^#?minlen', line: 'minlen = 14' }
        - { regexp: '^#?dcredit', line: 'dcredit = -1' }
        - { regexp: '^#?ucredit', line: 'ucredit = -1' }
        - { regexp: '^#?lcredit', line: 'lcredit = -1' }
        - { regexp: '^#?ocredit', line: 'ocredit = -1' }

    # ===== 文件权限 =====
    - name: 设置关键文件权限
      file:
        path: "{{ item.path }}"
        mode: "{{ item.mode }}"
        owner: root
        group: root
      loop:
        - { path: '/etc/passwd', mode: '0644' }
        - { path: '/etc/shadow', mode: '0600' }
        - { path: '/etc/group', mode: '0644' }
        - { path: '/etc/gshadow', mode: '0600' }
        - { path: '/etc/ssh/sshd_config', mode: '0600' }

    # ===== 内核安全 =====
    - name: 设置内核安全参数
      sysctl:
        name: "{{ item.key }}"
        value: "{{ item.value }}"
        sysctl_file: /etc/sysctl.d/99-security.conf
        reload: yes
      loop:
        - { key: "net.ipv4.conf.all.rp_filter", value: "1" }
        - { key: "net.ipv4.conf.default.rp_filter", value: "1" }
        - { key: "net.ipv4.conf.all.accept_redirects", value: "0" }
        - { key: "net.ipv4.conf.default.accept_redirects", value: "0" }
        - { key: "net.ipv4.conf.all.send_redirects", value: "0" }
        - { key: "net.ipv4.conf.default.send_redirects", value: "0" }
        - { key: "net.ipv4.conf.all.accept_source_route", value: "0" }
        - { key: "net.ipv4.conf.default.accept_source_route", value: "0" }
        - { key: "net.ipv4.conf.all.log_martians", value: "1" }
        - { key: "net.ipv4.icmp_echo_ignore_broadcasts", value: "1" }
        - { key: "net.ipv4.icmp_ignore_bogus_error_responses", value: "1" }
        - { key: "kernel.randomize_va_space", value: "2" }

    # ===== 禁用不需要的服务 =====
    - name: 禁用不需要的服务
      systemd:
        name: "{{ item }}"
        state: stopped
        enabled: no
      loop:
        - postfix
        - rpcbind
        - avahi-daemon
        - cups
      ignore_errors: yes

    # ===== 配置审计 =====
    - name: 配置 auditd 规则
      copy:
        dest: /etc/audit/rules.d/custom.rules
        content: |
          # 监控用户/组变更
          -w /etc/passwd -p wa -k identity
          -w /etc/group -p wa -k identity
          -w /etc/shadow -p wa -k identity
          -w /etc/gshadow -p wa -k identity
          
          # 监控 SSH 配置变更
          -w /etc/ssh/sshd_config -p wa -k sshd_config
          
          # 监控 sudo 配置
          -w /etc/sudoers -p wa -k sudoers
          -w /etc/sudoers.d/ -p wa -k sudoers
          
          # 监控计划任务
          -w /etc/crontab -p wa -k cron
          -w /var/spool/cron/ -p wa -k cron
          -w /etc/cron.d/ -p wa -k cron
          
          # 监控 root 命令执行
          -w /usr/bin/su -p x -k priv_escalation
          -w /usr/bin/sudo -p x -k priv_escalation

  handlers:
    - name: restart auditd
      command: service auditd restart
```

## 五、软件部署 Playbook

```yaml
# playbooks/03-software-deploy.yml
---
- name: 软件部署
  hosts: webservers
  become: yes
  serial: "30%"  # 滚动更新，每次更新 30%

  vars:
    app_version: "latest"
    app_port: 8080
    health_check_url: "http://localhost:{{ app_port }}/health"

  tasks:
    # ===== 前置检查 =====
    - name: 检查当前版本
      uri:
        url: "{{ health_check_url }}"
        return_content: yes
      register: current_health
      ignore_errors: yes

    # ===== 备份当前版本 =====
    - name: 备份当前版本
      archive:
        path: /opt/app
        dest: "/opt/backup/app-{{ ansible_date_time.iso8601_basic }}.tar.gz"
        format: gz
      when: current_health.status == 200

    # ===== 部署新版本 =====
    - name: 拉取新镜像
      docker_image:
        name: "harbor.example.com/production/app:{{ app_version }}"
        source: pull
        force_source: yes

    - name: 停止当前容器
      docker_container:
        name: app
        state: absent

    - name: 启动新容器
      docker_container:
        name: app
        image: "harbor.example.com/production/app:{{ app_version }}"
        state: started
        restart_policy: unless-stopped
        ports:
          - "{{ app_port }}:{{ app_port }}"
        env:
          APP_ENV: "production"
          APP_VERSION: "{{ app_version }}"
        volumes:
          - /opt/app/config:/app/config
          - /opt/app/logs:/app/logs
        resources:
          limits:
            cpus: "2"
            memory: 2G
        healthcheck:
          test: ["CMD", "curl", "-f", "http://localhost:{{ app_port }}/health"]
          interval: 30s
          timeout: 10s
          retries: 3

    # ===== 健康检查 =====
    - name: 等待服务启动
      uri:
        url: "{{ health_check_url }}"
        return_content: yes
      register: health_check
      until: health_check.status == 200
      retries: 30
      delay: 10

    # ===== 回滚机制 =====
    - name: 回滚到上一版本
      block:
        - name: 停止失败的容器
          docker_container:
            name: app
            state: absent

        - name: 启动备份版本
          docker_container:
            name: app
            image: "harbor.example.com/production/app:{{ app_previous_version }}"
            state: started
            restart_policy: unless-stopped
            ports:
              - "{{ app_port }}:{{ app_port }}"
      when: health_check.status != 200
      rescue:
        - name: 通知回滚
          debug:
            msg: "⚠️ 部署失败，已回滚到版本 {{ app_previous_version }}"
```

## 六、自动化巡检脚本

```bash
#!/bin/bash
# healthcheck.sh - 自动化巡检脚本
# 用法: crontab: 0 */6 * * * /opt/scripts/healthcheck.sh

set -euo pipefail

REPORT_DIR="/data/reports/healthcheck"
DATE=$(date +%Y%m%d_%H%M%S)
REPORT_FILE="${REPORT_DIR}/healthcheck-${DATE}.html"
ALERT_FILE="/tmp/healthcheck-alerts.json"

mkdir -p ${REPORT_DIR}

# 颜色输出
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log() { echo "[$(date '+%H:%M:%S')] $1" | tee -a ${REPORT_FILE}.log; }
warn() { echo -e "${YELLOW}[WARN]${NC} $1" | tee -a ${REPORT_FILE}.log; }
error() { echo -e "${RED}[ERROR]${NC} $1" | tee -a ${REPORT_FILE}.log; }
ok() { echo -e "${GREEN}[OK]${NC} $1" | tee -a ${REPORT_FILE}.log; }

# 检查项
declare -A RESULTS

# 1. 节点状态检查
check_node_status() {
  log "=== 节点状态检查 ==="
  
  for node in $(kubectl get nodes -o jsonpath='{.items[*].metadata.name}'); do
    status=$(kubectl get node ${node} -o jsonpath='{.status.conditions[?(@.type=="Ready")].status}')
    if [ "${status}" = "True" ]; then
      ok "节点 ${node}: Ready"
      RESULTS["node_${node}"]="ok"
    else
      error "节点 ${node}: NotReady"
      RESULTS["node_${node}"]="error"
    fi
  done
}

# 2. Pod 状态检查
check_pod_status() {
  log "=== Pod 状态检查 ==="
  
  # 检查异常 Pod
  bad_pods=$(kubectl get pods -A --field-selector=status.phase!=Running,status.phase!=Succeeded --no-headers 2>/dev/null | wc -l)
  
  if [ "${bad_pods}" -eq 0 ]; then
    ok "所有 Pod 正常运行"
    RESULTS["pod_status"]="ok"
  else
    error "发现 ${bad_pods} 个异常 Pod"
    RESULTS["pod_status"]="error"
    kubectl get pods -A --field-selector=status.phase!=Running,status.phase!=Succeeded
  fi
  
  # 检查重启过多的 Pod
  restart_pods=$(kubectl get pods -A -o json | jq -r '
    .items[] |
    . as $pod |
    .status.containerStatuses[]? |
    select(.restartCount > 5) |
    "\($pod.metadata.namespace)/\($pod.metadata.name): \(.restartCount) 次重启"
  ' 2>/dev/null)
  
  if [ -n "${restart_pods}" ]; then
    warn "以下 Pod 重启次数过多:"
    echo "${restart_pods}"
  fi
}

# 3. 资源使用检查
check_resource_usage() {
  log "=== 资源使用检查 ==="
  
  # CPU 使用率
  cpu_usage=$(kubectl top nodes 2>/dev/null | awk 'NR>1 {print $3}' | sed 's/%//' | head -1)
  if [ -n "${cpu_usage}" ] && [ "${cpu_usage}" -gt 80 ]; then
    warn "CPU 使用率过高: ${cpu_usage}%"
    RESULTS["cpu_usage"]="warn"
  else
    ok "CPU 使用率正常: ${cpu_usage:-N/A}%"
    RESULTS["cpu_usage"]="ok"
  fi
  
  # 内存使用率
  mem_usage=$(kubectl top nodes 2>/dev/null | awk 'NR>1 {print $5}' | sed 's/%//' | head -1)
  if [ -n "${mem_usage}" ] && [ "${mem_usage}" -gt 80 ]; then
    warn "内存使用率过高: ${mem_usage}%"
    RESULTS["mem_usage"]="warn"
  else
    ok "内存使用率正常: ${mem_usage:-N/A}%"
    RESULTS["mem_usage"]="ok"
  fi
  
  # 磁盘使用率
  disk_usage=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')
  if [ "${disk_usage}" -gt 80 ]; then
    warn "磁盘使用率过高: ${disk_usage}%"
    RESULTS["disk_usage"]="warn"
  else
    ok "磁盘使用率正常: ${disk_usage}%"
    RESULTS["disk_usage"]="ok"
  fi
}

# 4. 证书有效期检查
check_certificates() {
  log "=== 证书有效期检查 ==="
  
  # 检查 K8s 证书
  for cert in /etc/kubernetes/pki/*.crt; do
    expiry=$(openssl x509 -in ${cert} -noout -enddate 2>/dev/null | cut -d= -f2)
    expiry_epoch=$(date -d "${expiry}" +%s 2>/dev/null || echo 0)
    current_epoch=$(date +%s)
    days_left=$(( (expiry_epoch - current_epoch) / 86400 ))
    
    if [ "${days_left}" -lt 30 ]; then
      warn "证书即将过期: $(basename ${cert}), 剩余 ${days_left} 天"
      RESULTS["cert_$(basename ${cert})"]="warn"
    else
      ok "证书正常: $(basename ${cert}), 剩余 ${days_left} 天"
      RESULTS["cert_$(basename ${cert})"]="ok"
    fi
  done
}

# 5. etcd 健康检查
check_etcd() {
  log "=== etcd 健康检查 ==="
  
  etcdctl endpoint health \
    --endpoints=https://10.0.0.21:2379,https://10.0.0.22:2379,https://10.0.0.23:2379 \
    --cacert=/etc/kubernetes/pki/etcd/ca.crt \
    --cert=/etc/kubernetes/pki/etcd/etcd-client.crt \
    --key=/etc/kubernetes/pki/etcd/etcd-client.key 2>/dev/null
  
  if [ $? -eq 0 ]; then
    ok "etcd 集群健康"
    RESULTS["etcd"]="ok"
  else
    error "etcd 集群异常"
    RESULTS["etcd"]="error"
  fi
}

# 6. 服务健康检查
check_services() {
  log "=== 服务健康检查 ==="
  
  services=("nginx" "containerd" "kubelet" "kube-proxy")
  
  for svc in ${services[@]}; do
    status=$(systemctl is-active ${svc} 2>/dev/null || echo "unknown")
    if [ "${status}" = "active" ]; then
      ok "服务 ${svc}: 运行中"
      RESULTS["svc_${svc}"]="ok"
    else
      error "服务 ${svc}: ${status}"
      RESULTS["svc_${svc}"]="error"
    fi
  done
}

# 7. 安全检查
check_security() {
  log "=== 安全检查 ==="
  
  # 检查 SSH 配置
  root_login=$(grep -E "^PermitRootLogin" /etc/ssh/sshd_config | awk '{print $2}')
  if [ "${root_login}" = "no" ]; then
    ok "SSH: Root 登录已禁用"
    RESULTS["ssh_root"]="ok"
  else
    warn "SSH: Root 登录未禁用"
    RESULTS["ssh_root"]="warn"
  fi
  
  # 检查 SELinux
  selinux=$(getenforce 2>/dev/null || echo "unknown")
  if [ "${selinux}" = "Disabled" ] || [ "${selinux}" = "Permissive" ]; then
    ok "SELinux: ${selinux}"
    RESULTS["selinux"]="ok"
  else
    warn "SELinux: ${selinux}"
    RESULTS["selinux"]="warn"
  fi
}

# 生成报告
generate_report() {
  log "=== 生成巡检报告 ==="
  
  # 统计
  total=${#RESULTS[@]}
  ok_count=$(echo "${RESULTS[@]}" | tr ' ' '\n' | grep -c "^ok$" || true)
  warn_count=$(echo "${RESULTS[@]}" | tr ' ' '\n' | grep -c "^warn$" || true)
  error_count=$(echo "${RESULTS[@]}" | tr ' ' '\n' | grep -c "^error$" || true)
  
  cat > ${REPORT_FILE} <<EOF
<!DOCTYPE html>
<html>
<head>
  <title>巡检报告 - $(date '+%Y-%m-%d %H:%M')</title>
  <style>
    body { font-family: Arial; margin: 20px; }
    .summary { display: flex; gap: 20px; margin: 20px 0; }
    .card { padding: 20px; border-radius: 5px; text-align: center; }
    .ok { background: #d4edda; color: #155724; }
    .warn { background: #fff3cd; color: #856404; }
    .error { background: #f8d7da; color: #721c24; }
    .metric { font-size: 2em; font-weight: bold; }
  </style>
</head>
<body>
  <h1>巡检报告 - $(date '+%Y-%m-%d %H:%M')</h1>
  <div class="summary">
    <div class="card ok"><div class="metric">${ok_count}</div><div>正常</div></div>
    <div class="card warn"><div class="metric">${warn_count}</div><div>警告</div></div>
    <div class="card error"><div class="metric">${error_count}</div><div>异常</div></div>
  </div>
  <p>详细日志: ${REPORT_FILE}.log</p>
</body>
</html>
EOF
  
  echo ""
  echo "=== 巡检完成 ==="
  echo "报告: ${REPORT_FILE}"
  echo "总计: ${total} 项, 正常: ${ok_count}, 警告: ${warn_count}, 异常: ${error_count}"
  
  # 如果有异常，发送告警
  if [ "${error_count}" -gt 0 ]; then
    echo "⚠️ 发现异常，请检查报告"
    # 这里可以添加钉钉/企微通知
  fi
}

# 主函数
main() {
  log "=== 巡检开始: $(date '+%Y-%m-%d %H:%M:%S') ==="
  
  check_node_status
  check_pod_status
  check_resource_usage
  check_certificates
  check_etcd
  check_services
  check_security
  
  generate_report
}
```

## 八、面试题与思考

### Q1: Ansible 的幂等性是什么？如何保证？

**答：** 幂等性是指无论执行多少次，结果都相同。保证方式：

1. **使用声明式模块**：如 `yum`、`service`、`user`，它们会检查当前状态再决定是否执行
2. **使用 `creates`/`removes` 参数**：避免重复创建或删除
3. **使用 `when` 条件**：根据条件决定是否执行
4. **使用 `changed_when`**：自定义变更判断条件

### Q2: 如何实现 Ansible 的批量操作效率优化？

**答：**

1. **增加 forks**：调整 `forks` 参数，默认 5，可增加到 20-50
2. **使用 Pipelining**：启用 SSH pipelining，减少连接次数
3. **使用异步任务**：对于耗时操作，使用 `async` 和 `poll`
4. **分批执行**：使用 `serial` 控制批次
5. **缓存 facts**：使用 `gathering = smart` 缓存 facts
6. **使用 Mitogen**：使用 Mitogen 加速执行

### Q3: 如何设计 Ansible 的目录结构？

**答：** 推荐结构：

```
ansible-project/
├── ansible.cfg          # 项目配置
├── inventory/           # 主机清单
│   ├── production/
│   ├── staging/
│   └── dev/
├── playbooks/           # 剧本
│   ├── site.yml         # 主入口
│   └── *.yml            # 各功能剧本
├── roles/               # 角色
│   ├── common/
│   ├── nginx/
│   └── mysql/
├── group_vars/          # 组变量
├── host_vars/           # 主机变量
├── library/             # 自定义模块
├── plugins/             # 自定义插件
└── scripts/             # 辅助脚本
```

### Q4: Ansible 和 Puppet/Chef 有什么区别？

**答：**

| 特性 | Ansible | Puppet/Chef |
|------|---------|-------------|
| 架构 | 无 Agent，Push 模式 | 有 Agent，Pull 模式 |
| 语言 | YAML | DSL (Ruby) |
| 学习曲线 | 简单 | 较陡 |
| 部署 | 无需安装 Agent | 需要在每台机器安装 Agent |
| 适用场景 | 中小规模、临时任务 | 大规模、持续配置管理 |
| 商业支持 | Red Hat (AWX/Tower) | Puppet/Progress |

### Q5: 如何实现 CMDB 与 Ansible 的集成？

**答：** 集成方式：

1. **从 CMDB 导出 Inventory**：编写脚本从 CMDB（如 NetBox、iTop）导出主机信息到 Ansible Inventory
2. **自定义 Lookup 插件**：编写 Ansible Lookup 插件，直接从 CMDB 查询主机信息
3. **使用 AWX/Tower**：集成 CMDB 数据源，动态生成 Inventory
4. **回写执行结果**：Ansible 执行完成后，将结果回写到 CMDB
